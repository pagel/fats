Prerequisites

The FATS web application depends upon the following software:

  • PHP 5.3+ with PDO enabled
  • Smarty 3.1.2
  • Slim Framework 2.1.0

The repository contains the files necessary for Slim to function properly. Its notation above exists for informational purposes only.

Installation and Configuration

The setup process includes cloning the repository, setting Apache configuration options for the site, creating directories in specific locations, and setting application options.

Clone the Repository

In the Apache document root, clone the FATS repository.

$ git clone git@github.com:pagel/fats.git fats

Apache Configuration

For the application's web services to work properly, you must activate the FileInfo override for the site.

<Directory "/path/to/fats">
     AllowOverride FileInfo
     Order allow,deny
     Allow from all
</Directory>

Required Directories

The FATS web application is configured to store file uploads outside of the web root and downloads in the web root. These paths appear in src/DAL/DataAccessLayer.php and must be updated as appropriate to avoid issues with upload and download functionality.

The directories that store file uploads and downloads must be accessible to the process running httpd. Alternatively, you can set permissions such that any user can read and write to these directories.

$ chmod 777 /path/to/uploadordownload/directory

Application Options

The FATS web application stores certain pieces of information in an XML file for easy retrieval, including configuration options and database credentials. This file resides in /src/application.xml. If you choose to move or rename this file, you must update its location in /src/BLL/Application.php and /src/Helpers/ApplicationHelper.php to avoid error conditions.
